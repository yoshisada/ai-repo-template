PRDs generated by /kiln:kiln-distill should carry a `derived_from:` list in YAML frontmatter pointing at every source issue and feedback file that fed the PRD. Today the derivation is human-readable only — a `### Source Issues` markdown table in the PRD body. The reverse direction (issue → PRD) already has proper frontmatter (`prd: <path>`), but the PRD → issues direction requires parsing body markdown or doing an O(N) scan across `.kiln/issues/` + `.kiln/feedback/` matching the `prd:` field.

Why it's strategic, not tactical: the missing frontmatter is the root cause of a whole chain of string-matching complexity in `/kiln:kiln-build-prd` Step 4b. Step 4b's job is "given PRD path, find items derived from it." Today it inverts the map by string-matching `prd:` values across every backlog file — which is exactly the path-normalization bug we just shipped a fix for in PR #146. If the PRD instead carried `derived_from: [file1.md, file2.md]`, Step 4b would just read the PRD and iterate the list. No matching, no normalization, no diagnostic-on-zero-matches safety belt needed.

This also simplifies the hygiene rubric. The `merged-prd-not-archived` rule currently walks backlog files and does the same reverse-lookup dance. It could instead read the PRD's `derived_from` once and check archive status for each entry.

Scope of the proposal:
1. `/kiln:kiln-distill` Step 4 writes PRD frontmatter with `derived_from:` (list of relative paths), `distilled_date:`, `theme:`. The body `### Source Issues` table becomes a render of the frontmatter (single source of truth).
2. `/kiln:kiln-build-prd` Step 4b switches from scan-and-match to read-PRD-derived_from. The 6-field diagnostic from PR #146 stays for backwards compat / visibility but the match count should always equal the list length.
3. `/kiln:kiln-hygiene` `merged-prd-not-archived` rule switches to the same read-PRD-once shape.
4. Migration: one-time script or hygiene-style propose-a-diff that backfills `derived_from:` on existing PRDs by parsing their Source Issues table.

Area: architecture (affects distill's output contract + build-prd and hygiene consumers). Severity: medium — not blocking, but the string-match pattern keeps costing us real bugs (PR #146 was the second time this year).
