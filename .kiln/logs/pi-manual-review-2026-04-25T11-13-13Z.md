  Proposed: ADD a one-paragraph memo:
    "## Bash brace-pairing pitfall — `${var:-{}}` does NOT default to `{}`
    Bash parses `${var:-{}}` as `${var:-{`-then-`}}` — the FIRST `}` closes the parameter expansion, the second is a stray byte AFTER. To default to JSON empty-object literal, use `: \"${var:={}}\"` ONLY if the literal is single-character, or fall back to explicit conditional: `[[ -z \"$var\" ]] && var='{}'`. The same trap applies to any default ending in `}`. Discovered in wheel-typed-schema-locality during composer wiring (PR #168, agent-notes/implementer.md §2)."
  Why: Implementer flagged this as worth promoting. Bash brace-pairing is a recurring footgun across this codebase (jq -r format strings, JSON defaults). One-paragraph memo prevents re-discovery.
