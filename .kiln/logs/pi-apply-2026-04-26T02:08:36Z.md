## Proposed broader changes

1. **Decouple "audits complete" from "team-lead approves PR open" in build-prd team configs.** The PI-N-010 fix is the minimum; the broader pattern is that orchestrators should always have an explicit PR-open gate distinct from audit-complete signals so PAUSE-for-fix-up can interpose cleanly. Bake into the build-prd skill's team-config template, not just one-off in research-first-foundation.

2. **Standardize a "research-first" researcher role as a recurring pipeline pattern.** This pipeline added a `researcher-baseline` task that captured measurement-anchored NFRs before specifier ran. The aggregation-summary table + reconciliation handshake worked; promote this from one-off to canonical for any pipeline whose PRD has a measurement-grounded SC or NFR. Could be a `--research-first` flag on `/kiln:kiln-build-prd` that auto-injects the role.

3. **Audit-pr's PR template should be feature-flag aware.** When SC verdicts are mixed (some PASS, some FAIL with documented follow-ups), the template should default to the multi-verdict shape. Bake into `plugin-kiln/scaffold/.github/PULL_REQUEST_TEMPLATE.md` so consumers inherit it.

4. **Retention-on-fail should be the default trap pattern in kiln-test fixtures.** PI-N-009 fixes the specific instance; the broader rule is "any test that produces a diagnostic artifact (report, log, transcript) MUST retain it on failure." Add to test-fixture conventions doc.

5. **Reconciled-threshold tagging convention.** PI-N-005 specifier-side suggests `(RECONCILED 2026-04-25)` tag suffix on FRs/NFRs/SCs that were widened post-baseline. Combined with PI-N-003 (cross-artifact propagation), this gives reviewers a fast scan of which thresholds came from PRD literals vs measured baselines.

6. **`bashcov` decision should be revisited.** plan §Decision 5 deferred coverage instrumentation to "fixture suite as proof." The 5 fixtures cover the net-new code paths, but coverage is unmeasured. First implementation pipeline that touches research-runner.sh again should re-evaluate whether `bashcov` is easy to wire.
